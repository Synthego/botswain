# AWS Bedrock SDK Migration Summary

## Overview

Successfully migrated Botswain from subprocess-based Claude CLI to AWS Bedrock SDK with comprehensive token tracking and cost monitoring.

**Migration Date**: March 9-10, 2026
**Duration**: 2 days
**Status**: Complete ✅

## What Changed

### Architecture

**Before**: Used `subprocess` to call Claude CLI binary
- External process management overhead
- No built-in token tracking
- Limited error handling
- CLI dependency required on host

**After**: Direct AWS Bedrock SDK integration via `anthropic[bedrock]`
- Native Python SDK integration
- Automatic token counting and cost tracking
- Production-grade error handling
- Cloud-native scalability

### Key Improvements

1. **Reliability**: No subprocess overhead, direct API calls with proper exception handling
2. **Token Tracking**: Automatic capture of input/output tokens for every query
3. **Cost Monitoring**: Real-time cost calculation and reporting with USD estimates
4. **Scalability**: Better suited for production workloads with AWS infrastructure
5. **Configuration**: Environment-based configuration with sensible defaults
6. **Observability**: Complete audit trail with token usage metrics

## New Features

### Token Usage Tracking

All queries automatically track token consumption:

- **Input Tokens**: Tokens sent to the model (prompts, context)
- **Output Tokens**: Tokens generated by the model (responses)
- **Total Tokens**: Sum of input and output tokens
- **Storage**: Persisted in `QueryLog` model with new fields

**Database Fields Added**:
- `input_tokens` (PositiveIntegerField, nullable)
- `output_tokens` (PositiveIntegerField, nullable)
- `total_tokens` (PositiveIntegerField, nullable)
- `estimated_cost_usd` (DecimalField, 8 digits, 6 decimal places, nullable)

### Cost Reporting

New management command for analyzing token usage and costs:

```bash
# All-time report
python manage.py token_usage_report

# Date range filtering
python manage.py token_usage_report --start-date 2026-03-01 --end-date 2026-03-10

# User filtering
python manage.py token_usage_report --user username

# Makefile shortcuts
make token-report              # All-time
make token-report-today        # Today only
make token-report-week         # Past 7 days
make token-report-month        # Current month
```

**Sample Output**:

### Provider Factory Pattern

Easy switching between Bedrock and Claude CLI:

- **Environment Variable**: `LLM_PROVIDER=bedrock` or `claude_cli`
- **Factory Pattern**: `LLMProviderFactory.get_provider(provider_name)`
- **Default Provider**: BedrockProvider (production-ready)
- **Graceful Fallback**: ClaudeCLIProvider still available for development

### Cost Calculation

Real-time cost estimation using current AWS Bedrock pricing:

- **Sonnet 4.5 Pricing** (as of 2026):
  - Input: $3.00 per million tokens
  - Output: $15.00 per million tokens
- **Precision**: 6 decimal places (sub-cent accuracy)
- **Storage**: USD values stored in `estimated_cost_usd` field

## Known Limitations

1. **Cost Reporting Precision**: Management command displays costs with 2 decimal places for readability (e.g., "$3.42"), but database stores 6 decimal places for accuracy.

2. **Historical Data**: Existing QueryLog records (pre-migration) have NULL token/cost values since they used ClaudeCLIProvider without tracking.

3. **AWS Region**: Default is `us-west-2`, configurable via `AWS_REGION` environment variable. Ensure Bedrock is available in your chosen region.

4. **Model Access**: Requires explicit model access grants in AWS Bedrock console. Use inference profile IDs with `us.` prefix for cross-region availability.

5. **Cost Estimates**: Based on current pricing (March 2026). Actual costs may vary if AWS changes Bedrock pricing.

## Configuration Changes

### Required Environment Variables

```bash
# LLM Provider (required)
LLM_PROVIDER=bedrock

# AWS Region (optional, defaults to us-west-2)
AWS_REGION=us-west-2
```

### Optional Environment Variables

```bash
# Bedrock Model (optional, defaults to Sonnet 4.5)
BEDROCK_MODEL_ID=us.anthropic.claude-sonnet-4-5-20250929-v1:0

# Token Limits (optional, have sensible defaults)
BEDROCK_MAX_INTENT_TOKENS=500
BEDROCK_MAX_RESPONSE_TOKENS=1000

# Timeout (optional, defaults to 30 seconds)
BEDROCK_TIMEOUT=30.0
```

### AWS Credentials

Uses standard AWS SDK credential chain:
1. Environment variables (`AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`)
2. AWS credentials file (`~/.aws/credentials`)
3. IAM role (recommended for production)

## Migration Path

For existing deployments:

### Step 1: Update Dependencies
```bash
pip install -r requirements.txt
```

### Step 2: Run Migrations
```bash
python manage.py migrate
```

### Step 3: Configure Environment
```bash
export LLM_PROVIDER=bedrock
export AWS_REGION=us-west-2
```

### Step 4: Verify AWS Credentials
```bash
aws sts get-caller-identity
```

### Step 5: Test Configuration
```bash
make check-bedrock
```

### Step 6: Restart Application
```bash
make run
```

### Step 7: Monitor Token Usage
```bash
make token-report
```

**No code changes required** - migration is seamless via environment variables.

## Rollback Plan

If issues occur, rollback to Claude CLI:

```bash
# Change environment variable
export LLM_PROVIDER=claude_cli

# Restart application
make run
```

All existing functionality continues to work. Token tracking will be NULL for ClaudeCLIProvider queries, but queries execute normally.
